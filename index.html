우리의 미래: 실현 가능한 기술 아키텍처

.NET 생태계와 검증된 오픈소스를 중심으로, 데이터 주권과 통제권을 확보하는 구체적이고 실현 가능한 기술 청사진을 제시합니다.

🏛️ 수집 파이프라인 (Ingestion Pipeline)

다양한 소스로부터 데이터를 수집하여 우리의 핵심 자산인 **통합 벡터 DB**를 구축하고 최신 상태로 유지합니다.

1. 데이터 소스

MSSQL
파일 서버 (Excel, PDF, HWP)
ERP / CRM
↓

2. 데이터 수집

.NET Core Worker Service
MSSQL CDC: MSSQL 내장 기능인 Change Data Capture를 활성화하고, Worker Service가 주기적으로 변경분을 폴링하여 수집합니다.
File System Watcher: 지정된 파일 서버의 폴더를 모니터링하여 파일의 생성/수정 이벤트를 감지합니다.
API/DB Connector: ERP, CRM 등 내부 시스템의 데이터를 API 또는 DB Link를 통해 주기적으로 추출합니다.
→

3. 작업 오케스트레이션

Hangfire
수집, 처리, 임베딩 등 파이프라인의 모든 작업을 스케줄링하고, 종속성을 관리하며, 실패 시 재시도 정책을 수행하는 .NET 네이티브 오픈소스입니다. (Airflow의 .NET 버전)

Local 모델: 경량 모델을 ONNX 형식으로 변환하여, `Microsoft.ML.OnnxRuntime` 라이브러리를 통해 .NET 앱 내에서 직접 실행합니다. (CPU로도 충분히 빠름)
Cloud API: 필요시 OpenAI API를 호출하는 옵션도 병행 가능합니다.

6. 지식 저장

.NET Core에서 gRPC 또는 REST API 클라이언트를 통해 쉽게 연동할 수 있습니다.

🚀 추론 파이프라인 (Inference Pipeline)

사용자의 요청을 실시간으로 처리하며, **지능형 필터링 계층**을 통해 권한을 제어하고 벡터 DB를 참조하여 최종 답변을 생성합니다.

4. AI 모델

Local (Ollama) / OpenAI
증강된 프롬프트를 기반으로 최종 답변을 생성합니다.

Cloud API: OpenAI API를 호출하는 옵션도 선택 가능합니다.

🔄 CI/CD 및 운영 파이프라인 (CI/CD & MLOps)

코드 변경부터 배포까지 전 과정을 자동화하여 안정적이고 신속한 서비스 운영을 보장합니다.

1. CI (Continuous Integration)

Git Commit (GitHub)
개발자가 코드를 변경하고 GitHub 리포지토리에 푸시하면 파이프라인이 자동으로 시작됩니다.
↓
Build & Test (GitHub Actions)
Build: `dotnet build` 명령으로 .NET 프로젝트를 컴파일합니다.
Test: `dotnet test` 명령으로 유닛 테스트 및 통합 테스트를 실행하여 코드 품질을 검증합니다.
↓
Containerize (Docker)
테스트를 통과한 애플리케이션을 Dockerfile을 기반으로 컨테이너 이미지로 빌드합니다. 이를 통해 일관된 실행 환경을 보장합니다.
↓
Push to Registry (ACR)
빌드된 Docker 이미지를 버전 태그와 함께 Azure Container Registry(ACR)에 푸시하여 중앙에서 관리합니다.
→

2. CD (Continuous Deployment)

Deploy to Staging (AKS)
새로운 이미지를 스테이징 환경의 Azure Kubernetes Service(AKS) 클러스터에 배포합니다.
↓
Automated E2E Tests
스테이징 환경에서 실제 사용자의 시나리오를 모방하는 End-to-End 테스트를 자동으로 실행하여 기능의 안정성을 최종 검증합니다.
↓
Manual Approval
모든 테스트 통과 후, 담당자가 GitHub Actions/Azure Pipelines UI에서 최종 승인을 해야만 프로덕션 배포가 진행됩니다. (안전장치)
↓
Deploy to Production (AKS)
승인 후, Blue/Green 또는 Canary와 같은 무중단 배포 전략을 사용하여 프로덕션 환경에 안전하게 배포합니다.

자체 파이프라인의 전략적 가치

자체 파이프라인 구축은 단순한 비용 절감을 넘어, 비즈니스의 미래를 좌우할 핵심적인 전략적 이점을 제공합니다.

👑

데이터 주권 확보

가장 중요한 데이터 처리 과정과 그 결과물인 '벡터 DB'를 우리가 직접 소유하고 통제합니다. 데이터는 더 이상 외부에 종속되지 않는 우리의 완전한 자산이 됩니다.

🛡️

강력한 보안 및 권한 제어

우리가 직접 설계한 시스템 위에서만 부서/직급별 접근 제어(RBAC)와 민감정보 마스킹이 가능합니다. 이는 비즈니스의 신뢰와 직결되는 핵심 요소입니다.

🚀

미래 대응 유연성

금융 규제 변화로 Local LLM 도입이 필요해지거나, 더 뛰어난 AI 모델이 등장했을 때, 우리는 즉시 능동적으로 시스템을 교체하고 업그레이드할 수 있습니다.

🕊️

벤더로부터의 독립

업체를 바꾸더라도 우리의 핵심 자산인 벡터 DB는 그대로 유지됩니다. 우리는 언제든 최고의 기술을 가진 파트너와 자유롭게 협력할 수 있는 힘을 갖게 됩니다.

업무의 진화: 수동에서 자동화로

자체 파이프라인은 특정 담당자의 수작업에 의존하던 지식 관리 프로세스를 완전 자동화하여, 정보의 최신성을 극대화하고 운영 효율성을 혁신합니다.

AS-IS: 현재의 수동 프로세스

1. 수동 인지 및 취합

담당자가 신규 정보를 인지하고 직접 수집. 누락 위험 상존.

2. 수동 가공 및 전달

외부 업체 형식에 맞춰 수동으로 데이터 가공.

3. 업체 업로드 및 대기

업체 시스템에 업로드 후 반영까지 **수 시간 ~ 수 일 대기**.

🚨 문제점

  • 느린 반영 속도: 사용자는 낡은 정보로 답변받을 위험.
  • 인적 오류 발생: 데이터 누락 및 가공 오류 가능성.
  • 업무 병목 현상: 담당자 1인에게 모든 업무가 집중.

TO-BE: 자동화된 프로세스

1. 자동 감지 및 수집

시스템이 DB와 파일서버 변경을 **실시간으로 자동 감지**.

2. 자동 처리 및 벡터화

자동으로 문서를 파싱, 분할하고 벡터로 변환하여 메타데이터와 함께 저장.

3. 실시간 서비스 반영

내부 벡터 DB에 저장 즉시, **수 분 내로** 서비스에 반영.

💡 기대효과

  • 정보 최신성 유지: 항상 최신 정보를 기반으로 정확한 답변 제공.
  • 운영 효율성 극대화: 담당자는 더 가치 있는 업무에 집중.
  • 확장성 및 안정성: 새로운 데이터 소스도 손쉽게 연결 및 자동 처리.

미래를 향한 확장: AI 에이전트

구축된 파이프라인은 단순 질의응답을 넘어, 특정 과업을 자율적으로 수행하는 다양한 'AI 에이전트'를 신속하게 개발할 수 있는 강력한 플랫폼이 됩니다.

"AI 에이전트란, RAG 파이프라인, 외부 도구(API), 그리고 고유의 로직을 사용하여 특정 과업을 자율적으로 수행하는 특화된 AI 프로그램입니다."

- 예: 월간 실적 보고서 자동 생성, 계약서 독소 조항 검토 에이전트 등

1단계: 목표 정의 (Define the Goal)

에이전트의 명확한 임무(예: 월간 실적 보고서 생성)와 사용자, 입력, 출력(예: Markdown 보고서)을 구체화합니다.

2단계: 데이터 소스 및 도구 연결 (Connect)

필요한 데이터(예: MSSQL 실적 테이블)를 파이프라인에 연결하고, 사용할 외부 도구(예: ERP 조회 API)를 정의합니다.

3단계: 에이전트 로직 개발 (Develop)

정교한 프롬프트 엔지니어링과 ReAct(Reason+Act) 패턴을 적용하여, LLM이 스스로 생각하고 행동(API 호출 등)하며 과업을 수행하도록 로직을 개발합니다.

4단계: 통합 및 테스트 (Test)

개발된 에이전트를 메인 API 게이트웨이에 통합하고, 다양한 시나리오를 통해 안정성과 성능을 검증합니다.

5단계: 배포 및 모니터링 (Deploy)

수립된 CI/CD 파이프라인을 통해 새로운 에이전트를 안전하게 배포하고, 지속적인 모니터링과 사용자 피드백을 통해 고도화합니다.

단계별 실행 계획

리스크를 최소화하고 성공적인 전환을 위해, 핵심 기능부터 점진적으로 구축하고 내재화하는 3단계 실행 계획을 제안합니다.

Phase 1: 기반 다지기 - 데이터 파이프라인 구축 (3개월)

현재 업체에 제공하는 데이터를 그대로 활용하여, 내부적으로 데이터를 수집하고 정제하는 자동화된 파이프라인을 구축합니다. 이 단계의 목표는 데이터 처리의 '기반'을 마련하는 것입니다.

Phase 2: 핵심 자산 확보 - 벡터 DB 내재화 (2개월)

구축된 파이프라인을 통해 데이터를 벡터로 변환하고, 우리의 통제 하에 있는 벡터 DB에 저장합니다. 이 단계가 완료되면 우리는 핵심 지식 자산을 소유하게 됩니다.

Phase 3: 완전한 독립 - AI 모델 직접 연동 및 전환 (3개월)

내재화된 벡터 DB를 OpenAI 등 AI 모델과 직접 연동하여 기존 시스템과 동일한 성능을 내도록 구현합니다. 이후 점진적으로 외부 업체 시스템에서 내부 시스템으로 트래픽을 전환합니다.
